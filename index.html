Every system in Whiplash is hand-coded in C++. No marketplace assets. No default implementations. Purpose-built for performance, modularity, and AAA-grade gameplay feel.

01
CORE SYSTEM

CUSTOM GAMEPLAY
ABILITY SYSTEM

A lightweight, high-performance alternative to Unreal's default GAS. Tag-driven architecture using FGameplayTag structures for granting, executing, and revoking abilities with zero coupling.

  • Tag-driven ability grants & revocation
  • Attribute modifiers with delegate broadcasting
  • Abstract UWAbility base class for rapid prototyping
  • Isolated unit testing via AWAttributeSetTester

02
BALLISTICS ENGINE

ADVANCED WEAPON
FRAMEWORK

AAA-grade weapon system inspired by Epic's Lyra project. Procedural spread, first-shot accuracy, and material-based damage attenuation — all driven by data assets for designer-friendly iteration.

  • Real-time procedural spread & recoil
  • State-aware multipliers (ADS, crouch, airborne)
  • Distance & material-based damage falloff
  • Data-driven via UDataAsset configurations

03
MOVEMENT ENGINE

PARKOUR &
TRAVERSAL

Custom traversal component handling dynamic vault, hurdle, and mantle actions. Spline-based ledge detection with interface-driven obstacle math for seamless procedural animation playback.

  • Spline-based ledge detection & mapping
  • ITraversalObstacleInterface for hit math
  • Dynamic vault / hurdle / mantle classification
  • Gait-aware animation selection

Complete manual translation of Epic's Game Animation Sample from dense Blueprint graphs into clean, optimized native C++. Demonstrating deep understanding of both visual scripting and native engine code.

// UNDER THE HOOD

ENGINE-LEVEL
SYSTEMS

01

CVar Subsystem

Custom GameInstance Subsystem with macro-driven console variable interception. Hooks into IConsoleManager and broadcasts via Dynamic Multicast Delegates.

02

Tag-Based Physics

Extended UPhysicalMaterial with FGameplayTagContainer. Data-driven material interactions replacing hardcoded surface enums.

03

Memory Management

Heavy use of TWeakObjectPtr, TObjectPtr, and forward declarations. Zero circular dependencies, zero memory leaks.

04

Unit Testing

Dedicated AWAttributeSetTester actor for isolated system validation. Test-driven development approach for gameplay attributes.
